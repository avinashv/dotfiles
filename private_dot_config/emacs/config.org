* Philosophy

This config aims to *use Emacs natively*: built-in features over packages,
native key bindings (no =evil-mode=), =dired= over external file trees,
=tab-bar-mode= and =desktop-save-mode= over a separate multiplexer.

Non-native packages are added only where the built-in equivalent is
genuinely insufficient (=magit= for git, =vterm= for a real terminal,
=lsp-mode= for Mason-like server installation).

Loading is bootstrapped by =init.el=, which calls =org-babel-load-file= on
this file. After editing, restart Emacs or =M-x org-babel-load-file= the
config to pick up changes.

** Hide GUI chrome
Done per-frame so =emacsclient= frames pick up the same look.
#+begin_src emacs-lisp
(defun avi/clean-frame-ui (&optional frame)
  "Hide toolbars, scrollbars, menubar; quiet the cursor."
  (with-selected-frame (or frame (selected-frame))
    (when (display-graphic-p)
      (tool-bar-mode -1)
      (scroll-bar-mode -1)
      (menu-bar-mode -1)
      (set-fringe-mode 10))
    (blink-cursor-mode -1)))

(avi/clean-frame-ui)
(add-hook 'after-make-frame-functions #'avi/clean-frame-ui)
#+end_src

    "C-x 5"   "other-frame"
    "C-x 8"   "insert-special"
    "C-x w"   "window-extras"
    "C-x C-k" "keyboard-macros"

* Completion

** Minibuffer — fido-vertical (built-in)

Vertical fuzzy completion for =M-x=, =C-x C-f=, =C-x b=, and every other
=completing-read= prompt. Built-in successor to =ido-mode= (works
everywhere, not just files/buffers).

While at the minibuffer prompt:
- Type to narrow; matching is fuzzy (=flex=).
- =C-n= / =C-p= move through candidates.
- =RET= selects the *highlighted* candidate; =C-j= submits the *typed*
  text verbatim (useful when creating a new file).

#+begin_src emacs-lisp
(fido-vertical-mode 1)
(setq completion-styles '(flex basic)
      completion-category-overrides
      '((file (styles . (basic partial-completion flex)))))
#+end_src

** In-buffer — completion-preview (built-in, Emacs 30+)

Shows ghost-text completion suggestions inline as you type. Replaces
=company=. =TAB= accepts the suggestion; =M-n= / =M-p= cycle alternatives.

#+begin_src emacs-lisp
(when (fboundp 'global-completion-preview-mode)
  (global-completion-preview-mode 1))
#+end_src

* File management — dired (built-in)

Dired is Emacs's built-in file manager. Open with =C-x d=. Jump to the
current file's directory with =C-x C-j= (=dired-x=).

Inside dired:
| Key | Action                          |
|-----+---------------------------------|
| =RET= | open file/dir                 |
| =^=   | go to parent                  |
| =m=   | mark current line             |
| =u=   | unmark                        |
| =U=   | unmark all                    |
| =t=   | toggle marks                  |
| =D=   | delete marked                 |
| =R=   | rename / move                 |
| =C=   | copy                          |
| =+=   | mkdir                         |
| =g=   | refresh                       |
| =s=   | toggle sort by name/date      |
| =(=   | toggle long/short listing     |
| =M-x wdired-change-to-wdired-mode= | edit listing as text |

#+begin_src emacs-lisp
(use-package dired
  :ensure nil
  :commands (dired dired-jump)
  :config
  (require 'dired-x)                              ;; adds C-x C-j, dired-omit-mode
  (setq dired-dwim-target t                       ;; default copy/move target = other dired window
        dired-recursive-copies 'always
        dired-recursive-deletes 'always
        delete-by-moving-to-trash t)

  ;; macOS BSD ls lacks --group-directories-first; use coreutils' gls if installed.
  (if (and (eq system-type 'darwin) (executable-find "gls"))
      (progn
        (setq insert-directory-program "gls")
        (setq dired-listing-switches "-alh --group-directories-first"))
    (setq dired-listing-switches "-alh"))

  (add-hook 'dired-mode-hook #'auto-revert-mode))
#+end_src

** Tabs — tab-bar-mode (built-in)
Replaces tmux windows. Tabs hold their own *window layout*, not just a
single buffer.

| Key       | Action                                          |
|-----------+-------------------------------------------------|
| =C-x t 2= | new tab                                         |
| =C-x t 0= | close tab                                       |
| =C-x t o= | next tab                                        |
| =C-x t O= | previous tab                                    |
| =C-x t r= | rename tab                                      |
| =C-x t b= | switch to buffer in *new* tab                   |
| =C-x t t= | switch tab by name                              |
| =C-x t f= | open file in new tab                            |
| =C-x t d= | dired in new tab                                |
| =C-x t u= | undo tab close                                  |

* LSP — lsp-mode + lsp-ui

Kept (over the lighter built-in =eglot=) specifically for Mason-like
server installation: =M-x lsp-install-server= prompts for a server and
downloads it under =~/.emacs.d/.cache/lsp/=. Diagnostics flow through the
built-in =flymake=. Formatting is handled by =apheleia= (next section),
not the LSP server, so format-on-save is consistent across languages.

LSP prefix: =C-c l=. Inside an LSP-managed buffer:
| Key         | Action                              |
|-------------+-------------------------------------|
| =C-c l g g= | go to definition                    |
| =C-c l g r= | find references                     |
| =C-c l g i= | find implementations                |
| =C-c l g t= | go to type definition               |
| =C-c l h h= | hover documentation                 |
| =C-c l r r= | rename symbol                       |
| =C-c l a a= | code actions                        |
| =C-c l = b= | format buffer                       |
| =C-c l = r= | format region                       |
| =C-c l F a= | add workspace folder                |
| =C-c l F r= | remove workspace folder             |

Server management:
| Command                       | What it does                         |
|-------------------------------+--------------------------------------|
| =M-x lsp-install-server=      | Install a server (the Mason flow)    |
| =M-x lsp-update-server=       | Update an installed server           |
| =M-x lsp-uninstall-server=    | Remove an installed server           |
| =M-x lsp-doctor=              | Diagnose your LSP setup              |
| =M-x lsp-describe-session=    | Show what's running                  |

#+begin_src emacs-lisp
(use-package lsp-mode
  :hook ((python-mode python-ts-mode
          js-mode js-ts-mode
          typescript-mode typescript-ts-mode tsx-ts-mode
          go-mode go-ts-mode
          rust-mode rust-ts-mode
          c-mode c-ts-mode
          css-mode css-ts-mode
          web-mode
          lua-mode lua-ts-mode
          haxe-mode) . lsp-deferred)
  :commands (lsp lsp-deferred)
  :init
  (setq lsp-keymap-prefix "C-c l")
  :config
  (setq lsp-auto-guess-root t
        lsp-enable-suggest-server-download t
        lsp-headerline-breadcrumb-enable nil))

(use-package lsp-ui
  :after lsp-mode
  :config
  (setq lsp-ui-doc-enable t
        lsp-ui-sideline-enable t
        lsp-ui-sideline-show-hover nil))
#+end_src

* Terminal — vterm

Real terminal inside Emacs. With =vterm= + =tab-bar-mode= you do not need
=tmux=: each project gets its own toggle-able terminal buffer, and tabs
hold full window layouts.

| Key       | Action                              |
|-----------+-------------------------------------|
| =C-c v=   | open vterm in current window        |
| =C-c V=   | open vterm in other window          |
| =C-`=     | toggle project terminal popup       |
| =C-~=     | toggle, opening in current dir      |
| =C-c C-t= | toggle copy-mode (read/yank text)   |

Requires: =brew install cmake libvterm=.

#+begin_src emacs-lisp
(use-package vterm
  :bind (("C-c v" . vterm)
         ("C-c V" . vterm-other-window))
  :config
  (setq vterm-max-scrollback 10000))

(use-package vterm-toggle
  :bind (("C-`" . vterm-toggle)
         ("C-~" . vterm-toggle-cd))
  :config
  (setq vterm-toggle-fullscreen-p nil
        vterm-toggle-scope 'project))
#+end_src
